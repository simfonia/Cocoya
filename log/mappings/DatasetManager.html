Cocoya Dataset Manager 模組規範

本文件定義了 Cocoya 資料集管理器的核心 API、資料結構與視圖渲染邏輯。本模組採用核心引擎 (Spec) + 職責解耦組件架構。

1. 核心類別: DatasetSpec

位置: ui/src/modules/dataset_manager/spec.js

方法描述輸入/輸出
parseCSV(text) 靜態方法，將 CSV 字串轉換為物件陣列。 (String) -> Array<Object>
detectSchema(rows) 靜態方法，自動推斷欄位型別 (Int/Float/String) 並分配角色。 (Array) -> Object (Schema)
validate() 驗證目前 Spec 完整性。影像模式需有 label，表格模式需有 features。 () -> { ok, errors, warnings }
toJSON() 序列化為符合 NGX Spark 訓練格式的契約 JSON。 () -> Object

2. 全域 API: window.CocoyaDataset

位置: ui/src/modules/dataset_manager/index.js

API用途狀態
open() / close()開啟或關閉管理器 Modal。Stable
getCurrentSpec()獲取目前的 Spec 實體 (已同步 UI 狀態)。Stable
removeAnnotation(idx)在標註模式下移除特定索引的方框。Phase 4
refreshDynamicPanels() / refreshPreview()主動刷新資料管理器的動態面板與 Spec 預覽。Stable

3. 遠端對接與雲端 AI 傳輸 (Cloud Sync APIs)

本模組於「雲端 AI 模式」開啟且連線至遠端主機時，提供以下 Host/Webview 傳輸指令：

指令 (Command)傳輸方向用途與資料結構
checkRemoteEnvironment Webview ➔ Host (Extension) 觸發遠端 GPU 與 Docker 狀態檢查。
checkRemoteEnvironmentResult Host (Extension) ➔ Webview 回傳診斷數據：{ success, status: { cudaAvailable, gpuName, dockerRunning, gpuPassthrough, errors } }。
datasetUploadArchive Webview ➔ Host (Extension) 將本地資料集壓縮檔傳送至遠端：{ zipData: [Base64], projectName }。
datasetUploadResult Host (Extension) ➔ Webview 回傳於遠端寫入沙盒並調用 Python zipfile 原位解壓縮之結果：{ success, error }。

4. 視圖組件與標註互動

  • UIComponents (ui_components.js):
    • renderImageGrid: 高效渲染影像縮圖牆，支援 loading="lazy" 與點擊回調。
    • renderLabelStats: 渲染標籤樣本分佈統計，並基於名稱雜湊分配固定顏色。
  • UICanvas (ui_canvas.js):
    • 負責 Canvas 互動，支援 bbox (物件偵測方框) 與 line (自駕循線引導線) 兩種標註模式。
    • 雙模繪圖狀態機：在 line 模式下支援「滑鼠拖曳拉線」與「點擊起點再點擊終點」雙模。點擊起點後會進入 drawingState = 1 預覽拉伸線，再次點擊終點或距離大於 5px 的拖曳即可鎖定線段並產出。起點渲染為綠色圓圈，終點渲染為紅色圓圈並附帶前進方向箭頭。
    • 座標轉換與限幅 (Clamp): 一律將滑鼠像素座標轉換為 0.0 ~ 1.0 的比例座標 (Normalized Coordinates)。藉由 clamp(val, max) 將 mousedown/mousemove/mouseup 的滑鼠座標強制限幅在畫布解析度邊界內，防止拉出影像邊界時產出越界的比例座標。此機制一體適用於方框 (BBox) 與線段 (Line) 標註。
  • UI Layout (ui_layout.js):
    • 負責資料集管理器 Modal DOM 事件管理與視圖重構。
    • 防禦性清除與重置：為防止重複渲染「← 返回列表」按鈕，在 enterAnnotationMode 時會防禦性移除已存在的按鈕。同時點擊「清除資料」按鈕時會主動執行 exitAnnotationMode() 卸載畫布事件監聽並移除返回按鈕，確保狀態乾淨重置。
  • Importer (importer.js):
    • 使用 webkitdirectory 繞過路徑存取限制，並以父目錄名作為自動標籤來源。

5. 規範化路徑政策 (NGX Compatibility)

1. 專案名稱: 僅限 [a-zA-Z0-9_-]，中文將被自動過濾。
2. 資料目錄: 一律由系統自動管理為 `dataset/[projectName]/`，不開放手動修改。
3. 影像檔案: 採相對路徑存儲於 Spec，由 baseDir 提供根目錄。
4. 描述欄位: 完全支援中文與 Unicode。

6. 全域 SSH 帳密與對話框管理 API: window.CocoyaUI

位置: ui/src/ui/dialogs.js

本對話框為雲端 AI 模式所共享，暫存 SSH 連線資訊（不進行 localStorage 持久化以保證安全性），並提供全域輸入 Dialog 機制：

API / 屬性描述型別 / 簽章
sshConfig 全域暫存的 SSH 連線憑證。網頁 Reload 或刷新後重置為 null。 Object: { host, port, username, password }
showSshConfigDialog(onConfirm, onCancel) 強制彈出 SSH 帳密輸入對話框，自動設定 display = 'flex'，並在使用者確認或取消時回傳。 (Function, Function) -> void
ensureSshConfig(onConfirm, onCancel) 檢查 sshConfig 是否存在，若無則呼叫 showSshConfigDialog，若有則直接觸發 onConfirm。 (Function, Function) -> void
