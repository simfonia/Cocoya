SSH/SFTP API 對照表 (paramiko)

專案: Cocoya · 用途: 透過 SSH/SFTP 與 DGX Spark 遠端通訊 · 最後更新: 2026-06-24

1. SSH 連線與認證

功能寫法說明
建立 SSH 用戶端ssh = paramiko.SSHClient()
自動接受 host keyssh.set_missing_host_key_policy(paramiko.AutoAddPolicy())僅開發階段使用
連線ssh.connect(hostname, port, username, password, timeout=15)timeout 建議設 15 秒

2. 執行遠端指令 (SSH exec_command)

情境正確寫法說明
取得家目錄 ssh.exec_command('echo $HOME') 回傳如 /home/simfonia
檢查路徑 (~ 版本) ssh.exec_command('eval ls "~/path"') 需要 eval 展開 ~
檢查路徑 (絕對路徑) ssh.exec_command('ls "/home/user/path"') 最佳寫法
建立目錄 ssh.exec_command('mkdir -p "/abs/path"')
讀取指令輸出 stdout.read().decode('utf-8').strip()
檢查 exit status stdout.channel.recv_exit_status() 0 = 成功

3. SFTP 檔案傳輸

功能寫法說明
開啟 SFTPsftp = ssh.open_sftp()
檢查檔案存在sftp.stat(remote_abs_path)若不存在拋出 FileNotFoundError
上傳檔案sftp.put(local_path, remote_abs_path)
下載檔案sftp.get(remote_abs_path, local_path)
關閉 SFTPsftp.close()
❌ 重要陷阱：SFTP 不支援 ~ 路徑
sftp.stat('~/cocoya_ai/model.keras') → FileNotFoundError
必須先取得絕對路徑：ssh.exec_command('echo $HOME') → /home/user/cocoya_ai/model.keras → sftp.stat('/home/user/cocoya_ai/model.keras')

4. ZIP 打包注意事項

情境正確寫法錯誤寫法
ZIP 內路徑（不上層） arcname = os.path.relpath(filepath, DATASET_DIR) arcname = os.path.relpath(filepath, os.path.dirname(DATASET_DIR))
結果 ZIP 內：rock/001.jpg ZIP 內：gesture_dataset/rock/001.jpg（多一層）
