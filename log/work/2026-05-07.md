# Cocoya 工作日誌 (2026-05-07)

## 今日進度
### 1. 修復 Tauri 生產環境「Unexpected token '<'」錯誤
- **[問題診斷]**：發現 Vite 在打包時不會自動處理非 ESM 模組的靜態腳本（如 `utils.js`），導致 `dist/` 資料夾缺少 `src/` 與 `blockly/` 目錄。當 Webview 嘗試載入這些檔案時，後端回傳了 `index.html` (404 Fallback)，導致語法錯誤。
- [解決方案]：
    - **不搬移檔案**：為了維持與 VSIX (VS Code Extension) 的開發相容性，不採取將檔案移入 `public/` 的做法。
    - **Vite 自動同步**：新增 `ui/vite.config.js` 並實作 `copyCocoyaAssets` 外掛，在打包結束 (`closeBundle`) 時自動執行目錄複製。
- [路徑優化]：
    - 全面將 `index.html` 中的絕對路徑 (`/src/...`) 轉換為相對路徑 (`src/...`)。
    - 修正 `BridgeTauri` 中的 `mediaUri` 為 `src`，確保模組載入器能正確找到 `src/modules/` 下的定義與產生器。
