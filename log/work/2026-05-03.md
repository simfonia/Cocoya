# Cocoya 工作日誌 (2026-05-03)

## 今日進度
### 1. UI/UX 響應式佈局強化
- **[問題修正]** 解決 Tauri 模式下視窗寬度過小時，工具列左側群組（設定按鈕至最左方）溢出並疊加在右側代碼預覽標題區的問題。
- **[實作細節]**：
    - 修改 `ui/src/style.css`，將 `.toolbar-group` 的 `flex-wrap` 從 `nowrap` 改為 `wrap`，確保在極窄空間下按鈕能自動換行而非溢出。
    - 導入多層級響應式媒體查詢 (`@media`)：
        - 寬度 < 1000px：縮減路徑標籤 (`#file-breadcrumb`) 最大寬度至 180px。
        - 寬度 < 850px：縮減路徑標籤最大寬度至 120px，並壓縮間距。
        - 寬度 < 700px：隱藏路徑標籤，優先保留功能按鈕的可視性。
- [Tauri 配置]：
    - 更新 `src-tauri/tauri.conf.json`，加入 `minWidth: 960` 與 `minHeight: 600` 限制，從底層防止視窗縮小至無法使用的程度。

### 2. 程式碼預覽面板收合與縮放功能 (對齊 #wavecode)
- **[新功能]** 實作程式碼預覽面板的收合與展開功能，並支援手動調整垂直分割面積。
- **[實作細節]**：
    - **對齊 WaveCode 樣式**：
        - 在 `#blocklyArea` 與 `#codeArea` 之間新增 `#panel-resizer` (調整棒)，支援拖動縮放。
        - 在 `#codeArea` 左側邊緣新增隱藏式切換把手 (`#code-toggle`) 與動態箭頭，提供更直覺的收合體驗。
    - **縮放邏輯與效能優化**：
        - 在 `ui/src/ui_manager.js` 實作 `initLayout`，使用 `requestAnimationFrame` 優化 `mousemove` 頻率，大幅提升拖動流暢度。
        - **[關鍵修正]**：在拖動期間透過 CSS 強制禁用 `#codeArea` 的 `transition` 動畫，解決原本動畫延遲導致的 Blockly 工作區不對齊（灰色邊塊）問題。
        - **[安全機制]**：新增 `#resizer-overlay` 覆蓋層，防止拖動時滑鼠事件被 iframe 或 Blockly 內部組件截斷；並在 `mouseup` 時觸發最終 `svgResize` 確保佈局完全吻合。
        - 縮放時即時呼叫 `Blockly.svgResize` 確保積木區同步填滿。
        - 加入 `body.resizing-panel` 狀態類別，防止縮放時選取文字並維持滑鼠指標一致性。
    - **收合邏輯優化**：
        - 移除工具列上冗餘的「切換程式碼預覽」按鈕。
        - **[結構調整]**：將 `#code-toggle` 把手從 `#codeArea` 內部移至外部（作為同層兄弟節點），解決收合後因父容器 `opacity: 0` 導致把手消失的問題。
        - 點擊把手即可切換收合狀態，並同步更新箭頭方向：收合後顯示 `◀`（展開），展開後顯示 `▶`（收合）。
- **[i18n]**：
    - 補齊中英文語系檔中的 `TLB_TOGGLE_CODE` 字串與 Tooltip（雖然工具列按鈕已移除，但把手 title 仍在使用）。

- **[VSIX 模式修正]**：
    - 修正工具列「關閉編輯器」按鈕失效的問題，現已正確綁定 `close` 指令。
    - 在 VSIX 模式下自動隱藏「開啟/關閉終端機」按鈕，因為 VSIX 使用 VS Code 原生終端機，無需重複 UI。
    - 優化 `ui_manager.js` 的環境偵測邏輯，確保 Tauri 與 VSIX 模式下的按鈕顯示完全對齊需求。

## 備註
- 此修正同時兼顧了 Tauri 獨立版與 VS Code Webview 模式下的佈局穩定性。
- 路徑標籤的動態隱藏、面板收合與自由縮放功能，確保了即使在窄螢幕或分割視窗模式下，教學工具依然保有極佳的可用性。
