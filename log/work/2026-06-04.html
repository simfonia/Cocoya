技術深挖 (Technical Deep Dive)

1. 三層字串意識 (Three-Layer String Consciousness)

在處理 Blockly 產生器與產出的 Python/MicroPython 代碼時，我們必須注意三層轉義：

  • 第一層：Gemini API 傳輸的 Payload。
  • 第二層：寫入磁碟的產出器 .js 檔案。
  • 第三層：產出器執行後，回傳給 Blockly 的程式碼字串（最終寫入 .py 檔案）。

在 .js 檔案中定義字串時，若含有 \n（換行符號），必須寫為 \\n，以防它在寫入磁碟時即被解譯為實體換行，導致 JS 語法錯誤（實體換行在單引號或雙引號中是不合法的）。反之，若在 JavaScript 中使用反單引號 (Template Literals)，則允許合法的實體換行，這也是在注入長篇 Python 代碼時最安全的方式。

2. 正規表達式雙重轉義

當在 Java (PDE) 或 JavaScript 裡編寫 Regex（例如針對加號等特殊字元的 split），如果最終生成的代碼中需要反斜線（如 \\+），在產生器的 .js 中必須寫成四重轉義（\\\\\\\\+），才能確保在經歷多層轉換後依然正確解析為字面量。

3. 畫布座標限幅（Clamp）與防護設計

在使用者標註（BBox 拉框或 Line 畫線）過程中，若滑鼠移出影像區域外，容易產生負值或大於 1.0 的非法比例座標。我們藉由在 mousedown、mousemove、mouseup 統一將滑鼠事件座標（相對於 Canvas bounding rect）經過：

const clamp = (val, max) => Math.max(0, Math.min(max, val));

進行截斷防護，才進行後續的比例座標計算（如 x / canvas.width）。如此一來，在 BBox 及 Line 模式下均能自動確保標註座標與寬高永遠精確貼齊影像邊緣，比例座標必然落在 [0.0, 1.0] 閉區間內，保障了後續 TinyML 模型訓練資料集的純淨度與合規性。

4. 專案類型與來源模式之關聯狀態設計 (State Association Design)

原先系統在切換專案類型（如 Image 切換到 Table）時，僅重設了內部的資料陣列，並未同步關聯更新 DOM。由於 sourceMode 的 DOM 下拉選單依然殘留 live 值，當 UI 調用 syncSpecFromUI 與 refreshDynamicPanels 時，系統會誤判目前為 Live 模式，進而錯誤重啟攝影機與連拍。

重構後，我們定義了 TYPE_TO_MODES_MAP 靜態映射。在 typeSelect.onchange 時動態更新並限制 sourceMode 的 options 清單；若先前選取值不相容，則強制回滾為支援的預設模式（如 table 僅支援 file 模式）。如此一來，UI 的 DOM 狀態與內部的 DatasetSpec 狀態達成了原子化同步，避免了跨模式狀態混亂與背景資源洩漏。

5. 暫存與匯出路徑的 SSOT (單一事實來源) 設計

原先在 extension.ts 的 datasetCaptureImage 與 handleDatasetExport 裡，暫存和打包檔案所參考的 baseDir 預設採用 process.cwd()。在 VS Code 插件執行環境中，process.cwd() 常指向 VS Code 安裝路徑（例如 C:\Users\username\AppData\Local\Programs\Microsoft VS Code），這不但在某些系統環境有防寫限制，也容易因為執行期路徑改變而造成寫入路徑與打包路徑脫節。

本次修正為：若當前已開啟工作區 (Workspace)，一律使用工作區根目錄 vscode.workspace.workspaceFolders[0].uri.fsPath；否則降級使用擴充功能暫存目錄。此舉保證了補拍與匯出路徑的 SSOT 同步，且影像資料檔案均穩健儲存在使用者的工作區專案資料夾下，不再遺失。

6. Flexbox center overflow 滾動失效原理

在 CSS Flexbox 佈局中，如果容器被設定為 flex-wrap: wrap; 且帶有垂直置中（如 align-content: center; 或 align-items: center;），一旦子元素的總高度大於容器的最大高度 (max-height) 產生溢出時，瀏覽器會把內容往上下兩個方向均等推開。這會導致位於上半部分的項目被推至容器邊界（即負座標區）之外。由於瀏覽器捲軸只能向正方向捲動，無法向負方向捲動，使用者將永遠無法透過滾輪或滾動條捲動到頂部的內容。將對齊改為 flex-start; 即可使溢出內容往單一方向排列，完美回復正常滾動。
