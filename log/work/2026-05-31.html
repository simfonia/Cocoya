技術深挖 (Technical Deep Dive)

1. 標籤持久化策略 (Label Map Persistence)

原先 updateStatsFromImages 會根據影像樣本重新計算標籤清單，導致剛新增但無樣本的標籤遺失。現改採 Object.assign({}, existingLabelMap) 合併偵測到的標籤，確保 UI 新增的空類別能穩定存續。

2. HSL 配色之視覺辨識度

捨棄靜態 Hex 配色，改用 hsl(${h}, 75%, 45%)。透過控制飽和度 (Saturation) 與亮度 (Lightness)，確保自動生成的顏色在深色/淺色模式下均具備極高的對比度與辨識度。

3. Sidecar stderr 分流機制

Python Sidecar 將非 JSON 日誌輸出至 stderr 是為了保持 stdout 協議純淨。Extension 端現在會識別 [Sidecar Log] 標籤進行分流，避免 VS Code 將開發日誌誤判為程式崩潰。

4. MicroPython 函式作用域與 Global 衝突

在 Python 中，若函式內包含對變數的賦值（如 var = ...），該變數會被視為區域變數。原先 MicroPython 產生器的 Lazy Init 寫法 if 'var' not in globals(): var = ... 在變數已存在於全域時會跳過賦值，導致後續程式碼存取區域變數時拋出 UnboundLocalError。現統一改為 if 'var' not in globals(): globals()['var'] = ... 後接 var = globals()['var'] 之安全模式，確保在全域與區域作用域下皆能正確存取實例。
