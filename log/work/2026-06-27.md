## 問題根因分析

### Toolbox 分類標籤顯示 `%{BKY_CAT_AI_INFERENCE}`

**根因**：`ai_inference/i18n/zh-hant.js` 和 `en.js` 中定義了錯誤的 key：
```js
Blockly.Msg["BKY_CAT_AI_INFERENCE"] = "AI 推論";  // ❌ 多加了 BKY_ 前綴
```

Blockly 核心解析 `%{BKY_XXX}` 佔位符的機制（見 `ui/blockly/core/blockly.js`）：
```
%{BKY_CAT_AI_INFERENCE}
  → 提取 key: BKY_CAT_AI_INFERENCE
  → 自動去掉 BKY_ 前綴: CAT_AI_INFERENCE
  → 查詢 Blockly.Msg["CAT_AI_INFERENCE"] → 找不到 ❌
```

其他 AI 模組（ai_face、ai_hand、ai_pose）正常是因為全域 `zh-hant.js` 中定義了：
```js
"CAT_AI_FACE": "臉部網格"  // 無 BKY_ 前綴
```

而 `CAT_AI_INFERENCE` 在全域中完全不存在。

**修正**：
1. `ai_inference/i18n/zh-hant.js` 第3行：`Blockly.Msg["BKY_..."]` → `Blockly.Msg["CAT_AI_INFERENCE"]`
2. `ai_inference/i18n/en.js` 第3行：同上
3. `toolbox.xml` 第1行：`colour="%{COLOUR_AI_INFERENCE}"` → `colour="%{BKY_COLOUR_AI_INFERENCE}"`（補上 BKY_ 前綴）
4. `ui/src/zh-hant.js`：補上 `"CAT_AI_INFERENCE": "AI 推論"` 和 `"COLOUR_AI_INFERENCE": "#9C27B0"`
5. `ui/src/en.js`：同上英文版

### Toolbar 關閉按鈕顯示 `%{BKY_TLB_CLOSE_EDITOR}`

## Technical Deep Dive

### Blockly i18n 佔位符解析機制

Blockly 核心程式碼位於 `ui/blockly/core/blockly.js`（minified）：

```javascript
// 解析 tokenizeInterpolationInternal 流程
h=f.toUpperCase()
h=h.startsWith("BKY_") ? h.substring(4) : null  // 自動去掉 BKY_ 前綴
h in $.Msg  // 查詢 Blockly.Msg[XXX]（無 BKY_ 前綴）
```

**關鍵規則**：
- `toolbox.xml` / `blocks.js` 中的佔位符：必須使用 `%{BKY_XXX}`（有 `BKY_` 前綴）
- `Blockly.Msg` 定義：必須使用 `Blockly.Msg["XXX"]`（無 `BKY_` 前綴）

### 正確認的 i18n 寫法

```xml
<!-- toolbox.xml - 有 BKY_ 前綴 -->
<category name="%{BKY_CAT_AI_INFERENCE}" colour="%{BKY_COLOUR_AI_INFERENCE}">
```

```javascript
// 模組 i18n - 無 BKY_ 前綴，Object.assign 風格
Object.assign(Blockly.Msg, {
  "CAT_AI_INFERENCE": "AI 推論",
  "COLOUR_AI_INFERENCE": "#9C27B0"
});
```

```javascript
// 全域 i18n - 無 BKY_ 前綴，Object.assign 風格
Object.assign(Blockly.Msg, {
  "CAT_AI_INFERENCE": "AI 推論",
  "COLOUR_AI_INFERENCE": "#9C27B0"
});
```

## 知識蒸餾

### Blockly i18n 規範
- **`%{BKY_XXX}` 佔位符**：在 XML/JSON 中使用，必須有 `BKY_` 前綴
- **`Blockly.Msg["XXX"]` 定義**：在 JS 中使用，必須無 `BKY_` 前綴
- **全域 fallback**：所有 category name 和 colour 在全域 `zh-hant.js`/`en.js` 中定義一份無 `BKY_` 前綴的版本，作為模組 i18n 載入失敗時的 fallback
- **風格統一**：所有模組 i18n 使用 `Object.assign(Blockly.Msg, {...})` 風格（風格A）
