技術深挖

SSH exec_command 的 unzip_cmd 必須是單行指令。triple-quote 多行字串在 bash 中換行被視為命令結束符，導致 python3 -c 只接收到第一行而失敗。改用單行字串拼接後，bash 能完整解析整條命令。

技術深挖 (Technical Deep Dive)

之前的「無動靜、無報錯」是由於 VS Code 後端與 Sidecar 溝通時，若 spawn 進程失敗（例如本地找不到 Python、或者路徑無效），DatasetSidecarManager.send() 函數直接 return 中斷執行，導致前端的 callback 永遠不會被呼叫。在這種情況下，Webview 的 loading 與 diagnostics 狀態不會改變，亦無 any 錯誤訊息。 此外，對話框在 DOM 中已成功建立，但由於 CSS 類別 .dataset-manager-overlay 預設了 display: none，使得動態建立的 SSH 輸入框在沒有被顯式設為 display: flex 時被保持隱藏。這兩點交互作用造成不論何處操作皆無反應。 我們將其改為：若啟動失敗，則在 send() 內建構一個虛擬的失敗回應並執行 callback，讓前端能顯示「無法啟動本地 Python Sidecar...」的提示。 同時在 dialogs.js 中手動為對話框設定 display = 'flex'。 另外，將所有 Stdout/Stderr 輸出接引到自訂的 OutputChannel，可讓開發者在 VS Code 的「輸出」面板中追蹤 paramiko 與 SFTP 的即時過程，提高系統可觀測性。
