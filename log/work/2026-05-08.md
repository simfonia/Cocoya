# Cocoya 工作日誌 (2026-05-08)

## 今日進度
### 1. 完善 Tauri 建置與部署流程
- **[路徑修復]**：修正 `BridgeTauri` 中的 `mediaUri` 為 `src`，確保模組載入器在生產環境中能正確找到 `src/modules/` 下的定義與產生器。
- **[打包優化]**：在 `.vscodeignore` 中加入 `ui/dist/`，確保 VSIX 打包時排除 Tauri 產出的冗餘資產。
- **[安裝體驗]**：調整 `tauri.conf.json` 中的 NSIS 配置，將 `installMode` 設為 `both`，使安裝程式能同時支援「免權限 (Per-User)」與「全域 (Per-Machine)」模式，提升學校環境相容性。

### 2. Rust 後端模組化重構 (Major Refactor)
- **[結構優化]**：將單一巨大的 `lib.rs` 拆分為邏輯清晰的子模組：
    - `state.rs`：AppState 狀態定義。
    - `utils.rs`：跨平台路徑解析工具。
    - `commands/python.rs`：Python 執行與環境檢查。
    - `commands/file.rs`：檔案管理、鎖定與備份邏輯。
    - `commands/mcu.rs`：硬體通訊、韌體燒錄與序列埠偵測。
    - `commands/app.rs`：視窗管理與全域資訊。
- **[SSOT 建立]**：建立 `docs/backend_api_manifest.md` 作為後端 API 的唯一事實來源，與前端 API 清單對齊。
- **[穩定性修正]**：修正了重構過程中出現的 `Emitter`/`Manager` trait 缺失問題，並解決了進程清理時的型別推導錯誤。
