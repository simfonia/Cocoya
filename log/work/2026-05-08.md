# Cocoya 工作日誌 (2026-05-08)

## 今日進度
### 1. 完善 Tauri 建置與部署流程
- **[路徑修復]**：修正 `BridgeTauri` 中的 `mediaUri` 為 `src`，確保模組載入器在生產環境中能正確找到 `src/modules/` 下的定義與產生器。
- **[打包優化]**：在 `.vscodeignore` 中加入 `ui/dist/`，確保 VSIX 打包時排除 Tauri 產出的冗餘資產。
- **[安裝體驗]**：調整 `tauri.conf.json` 中的 NSIS 配置，將 `installMode` 設為 `both`，使安裝程式能同時支援「免權限 (Per-User)」與「全域 (Per-Machine)」模式，提升學校環境相容性。

### 2. Rust 後端模組化重構 (Major Refactor)
- **[結構優化]**：將單一巨大的 `lib.rs` 拆分為邏輯清晰的子模組（`state.rs`, `utils.rs`, `commands/`）。
- **[SSOT 建立]**：建立 `docs/backend_api_manifest.md` 作為後端 API 的唯一事實來源。
- **[穩定性修正]**：修正了 `Emitter`/`Manager` trait 導入與型別推導錯誤。

### 3. 前端「解耦重構」 (Environment Agnostic Refactor)
- **[能力系統]**：實作 `Capabilities` 系統，透過 `Bridge` 層抽象化環境功能差異（如 `hasTerminal`, `canClose`）。
- **[中央控制]**：建立 `AppController` 中央化指令轉發機制，消滅 `lifecycle.js` 中的巨大 `switch` 分支。
- **[純淨化 UI]**：`ui/base.js` 改依賴能力清單顯示按鈕；`persistence.js` 移除直接 DOM 操作，達成核心業務邏輯與執行環境的 100% 解耦。
