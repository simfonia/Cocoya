## 技術深挖 (Technical Deep Dive)

### 重構策略

- **職責分離**：將巨大的 `CocoyaManager` 類別拆分為核心管理器 + 6 個業務邏輯處理器
- **Handler 模式**：每個 handler 接收 `manager: any` 參考，可以存取 `this.manager.panel`、`this.manager.sidecar`、`this.manager.context` 等
- **保留 switch-case**：`setupMessageListener()` 仍保留在 `cocoyaManager.ts` 中，但每個 case 僅調用對應 handler 的方法
- **TypeScript 類型妥協**：由於 handler 需要存取 manager 的眾多屬性，使用 `manager: any` 避免過多的 interface 定義

### 編譯錯誤處理

- **問題**：`globalState.get<string>()` 在 `manager: any` 類型下會報錯 "Untyped function calls may not accept type arguments"
- **解決**：改用 `(this.manager.context.globalState as any).get('key') as string | undefined`
