# 2026-05-04 工作日誌

## 本日工作重點
- **[重大修復] 徹底解決 Tauri 視窗關閉攔截失效問題**：
    - **權限補全**：修改 `src-tauri/capabilities/default.json`，將視窗權限由 `["main"]` 擴展至 `["*"]`，並新增 `core:window:allow-destroy` 與 `core:window:allow-close`。
    - **標題數據流同步**：修正 `bridge.js` 中的 `setWindowTitle`，確保原生標題與 `document.title` 同步，解決 JS 攔截器偵測不到星號 (`*`) 的問題。
    - **穩定性加固**：將 `onCloseRequested` 監聽器改回同步判定，並在 `main.js` 引入 `isInitializing` 旗標與延遲監聽機制，解決啟動時誤判 Dirty 的 Bug。
- **[優化] 多視窗備份隔離與自動清理**：
    - **視窗隔離**：在 Rust 後端實作 `untitled_backup_{label}.xml` 命名機制，利用視窗 Label 隔離不同專案的自動備份。
    - **自動清理**：修改 `save_file` 指令，確保手動存檔成功後立即主動刪除該視窗對應的所有備份檔，防止重啟後出現過期的恢復提示。
- **[重大修復] 解決 VSIX 模式下 Python 環境檢查卡住與偵測不準確問題**：
    - **非同步重構**：將 `extension.ts` 中的環境檢查由同步 `execSync` 改為非同步 `execFile`，徹底解決 Extension Host 卡死問題。
    - **指令轉義優化**：透過 `execFile` 以數組傳遞參數，避開 Windows Shell 的引號轉義陷阱，確保套件偵測 100% 準確。
    - **合併檢查腳本**：將多個套件檢查合併為單次 Python 呼叫，大幅提升回應速度。
- **[清理] 移除冗餘日誌**：清理了 `main.js` 與 `bridge.js` 中所有偵錯用的 `console.log` 與 `console.error`。
