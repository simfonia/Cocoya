🔧 技術深挖 (Technical Deep Dive)

1. DGX Spark ARM64 架構的 Docker 相容性

  • DGX Spark 使用 linux/arm64（非一般桌機的 AMD64）
  • TensorFlow 官方 Docker 映像無 ARM64 GPU 版本，改用 nvcr.io/nvidia/pytorch:25.04-py3（NGC ARM64 容器）
  • TFLite 轉換在 ARM64 上有 keras_deps.get_call_context_function() bug，需在本地 AMD64 進行

2. SSH/SFTP 跨平台路徑處理

  • os.uname().nodename 在 Windows 不存在 → 改用 socket.gethostname()
  • SFTP 不支援 ~ 路徑展開 → 先透過 ssh.exec_command('echo $HOME') 取得絕對路徑
  • SSH exec_command 中的 ~ 需要 eval 展開

3. labels.txt 變數覆蓋 bug

  • Class Weight 計算的 for 迴圈 for _, labels in train_ds: 覆蓋了原有的 label 字串陣列
  • 修復：改用 for _, label_batch in train_ds:

4. 訓練最佳實踐

  • 驗證 加入 none/ 背景類別可顯著降低誤判率
  • 驗證 Class Weight 有效解決資料不平衡（none 32 張 vs paper 148 張）
  • 驗證 Data Augmentation 加強版（RandomBrightness + RandomTranslation）改善泛化能力
  • 最終成果：本地 92.55% → DGX 94.68%（30 epochs, 4 分類）
