## 技術深挖 (Technical Deep Dive)

### 靜態 ESM 載入決策
- Cocoya 既有 `bridge.js` 與 `main.js` 已使用 ESM，且 VSIX `extension.ts` 會將 `src` / `href` 轉為 webview URI。
- Dataset Manager 不屬於 Blockly toolbox 模組，因此不應加入 `core_manifest.json`。
- 使用靜態 ESM 可避免第一版就碰到動態 `import()`、CSP 與 VSIX Webview 路徑權限問題，同時保留標準 `import/export` 的模組化能力。

### Schema 偵測策略
- `detectSchema(sampleRows)` 以 row object 陣列為輸入，掃描所有欄位名稱並推斷型別。
- 型別推斷順序：
  1. boolean
  2. int
  3. float
  4. image_path
  5. timestamp
  6. string
- 多筆資料若混合 `int` 與 `float`，合併為 `float`；若混入 string，保守合併為 `string`。
- 欄位名稱為 `label`、`class`、`target` 時自動標記為 label。
