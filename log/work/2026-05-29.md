## 技術深挖 (Technical Deep Dive)

### 1. 影像匯入名稱填充與 Schema 同步修正
- **問題 A**：匯入 CSV 後欄位列表空白。原因是 `syncSpecFromUI` 在 DOM 尚未準備好前執行，導致偵測到的 Schema 被空 UI 覆蓋。
- **問題 B**：中文資料夾名稱無法自動填充。原因是 `sanitizeName` 過於嚴格且 fallback 邏輯未考慮視覺回饋。
- **解決方案**：
    - 調整執行順序，確保先渲染 DOM 再執行同步。
    - 強化 `buildLabelMap` 的保護機制，在未選定標籤欄位時保留偵測結果。
    - 允許非英數名稱填充至 UI 供使用者參考（但保留 validation 警告）。

### 2. 大數據量輸入效能優化 (Input Lag Fix)
- **問題**：當資料集包含數千張影像時，在 Modal 內打字會極度卡頓。
- **分析**：
    - 每次 `input` 事件都會觸發 `syncSpecFromUI`，其中的 `state.images.map` 是 O(N) 操作。
    - `JSON.stringify` 處理數千筆樣本路徑導致主執行緒阻塞。
- **解決方案**：
    - **防抖 (Debounce)**：將 `refreshPreview` 的重度任務延遲 300ms 執行。
    - **輕量化同步**：在打字期間僅同步基本屬性，跳過樣本清單的重建。
    - **預覽截斷**：在 JSON 預覽區僅顯示前 20 筆樣本，防止 DOM 渲染過長字串導致的卡死。
