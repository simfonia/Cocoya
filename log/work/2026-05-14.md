# Cocoya 工作日誌 (2026-05-14)

## 今日進度
### 1. 核心產生器重構 (Blockly Generator Refactor)
- **[解耦定義與執行]**：重構 `py_function_def`、`py_import` 與 `py_import_from` 產生器。
    - **改動**：利用 `generator.definitions_` 存儲定義，取代原本的 Statement 直接回傳。
    - **效果**：確保所有函式與庫引用在產出的 Python 程式碼中永遠排在主程式流之前，徹底解決因積木擺放高度導致的 `NameError`。
- **[修復硬體作用域 Bug]**：修正 `mcu_car` 模組中馬達與舵機初始化邏輯。
    - **問題**：在函式內使用硬體積木時，內部的 `if 'var' not in globals(): var = ...` 賦值會導致 Python 誤判為區域變數。
    - **修復**：將初始化程式碼移至 `definitions_` 全域定義區，確保函式內部可安全存取全域硬體物件。

### 2. UI 語意與術語優化
- **[更名：Local -> Parameter]**：將 Functions 分類下的「區域變數 (local)」更名為更直覺的「參數 (parameter)」。
    - **同步範圍**：包含積木標籤、Tooltip 說明文字、i18n Key 值 (`PY_LOCAL` -> `PY_PARAMETER`)。

### 3. 雲端 AI 模型訓練整合 (第一階段：環境感知)
- **[UI 模式切換]**：在工具列右上角新增「雲端 AI 模式」開關，具備霓虹燈視覺反饋。
- **[跨端連線感知]**：
    - **Extension 層**：實作 `vscode.env.remoteName` 偵測，區分本地開發與 Remote-SSH 環境。
    - **Bridge 層**：重構 `BaseBridge` 使 `capabilities` 支援動態更新，實作 Host 端與 Frontend 端的能力同步鏈。
- **[狀態持久化修復]**：修正啟動時雲端模式狀態無法維持開啟的 Bug，確保 `manifestData` 同步酬載包含 `cloudAiEnabled`。
- **[遠端路徑隔離 (Cloud Sandboxing)]**：實作 `getMachineId` (Hostname) 並在雲端伺服器自動初始化 `~/cocoya_ai/sessions/[MachineID]/` 隔離目錄。

### 4. 開發環境優化
- **[TS 類型修正]**：更新 `tsconfig.json` 補齊 Node.js 類型定義與 `esModuleInterop`。
- **[VSIX 遠端偵錯]**：配置 `package.json` 中的 `extensionKind: ["ui"]`，確保插件在 SSH 遠端視窗中維持可見。
