### 2. [資料集管理器] Phase 3: 資料匯入器 (Importer) 與 欄位對照 (Field Mapper)
- **目標**：實現 CSV/JSON 檔案的智慧解析與自動 Schema 生成。
- **成果**：
    - **CSV 解析**：在 `spec.js` 實作 `DatasetSpec.parseCSV`，支援基礎數值型別轉換。
    - **智慧偵測**：匯入檔案後自動執行 `detectSchema`，推斷欄位型別並分配角色 (Feature/Label)。
    - **預覽視圖**：在 UI 左側實作前 5 筆資料的表格預覽，方便即時比對。
    - **雙向同步**：調整 UI 欄位後，右側 JSON 預覽與 `DatasetSpec` 實體同步更新。

### 4. [資料集管理器] Phase 4: 影像匯入與互動標註系統
- **目標**：實現影像目錄批次匯入、自動標籤提取與 Bounding Box 互動標註。
- **成果**：
    - **目錄解析**：實作 `importer.js`，利用 `webkitdirectory` 支援資料夾批次匯入，並自動解析子目錄名作為 Label。
    - **動態面板**：重構 `ui_layout.js` 佈局。當專案類型切換時，自動在「表格預覽」與「影像縮圖牆」之間切換。
    - **標註系統**：實作 `ui_canvas.js` 互動畫布，支援滑鼠拖拽拉框標註，座標自動轉換為 0~1 比例系統並即時同步至 JSON。
    - **規範化限制**：為確保雲端 AI 訓練 (NGX) 相容性，實作了專案名稱與目錄欄位的「僅限英數下劃線」限制。
- **優化與修復**：
    - 解決影像模式下 JSON 預覽被覆蓋的問題。
    - 解決 `<input type="file">` 重複選擇同路徑不觸發的問題（匯入後重置 value）。
    - 解決筆電小螢幕 (Height < 800px) 下 Modal 顯示不全的問題，優化為自動全螢幕模式。

## 技術深挖 (Technical Deep Dive)
- **路徑安全原則**：確立「系統名稱英數化、說明描述中文語」的原則。專案名稱若含中文則自動過濾，確保與後端 Python 路徑拼接時 100% 相容。
- **畫布座標系統**：標註採用比例座標 (Normalized Coordinates)，在 `ui_canvas.js` 中動態換算，解決不同預覽尺寸下的標註位移問題。
