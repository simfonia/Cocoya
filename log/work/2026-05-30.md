## 任務摘要

## 技術深挖 (Technical Deep Dive)

### 1. Dataset Runtime Sidecar 架構轉型
- **動機**：解決 VS Code Webview 封鎖 `getUserMedia` 導致無法進行攝影機預覽與高品質擷取的問題。
- **解決方案**：
    - **Backend (Python)**：建立一個獨立的 Python Sidecar。使用 OpenCV 開啟原生系統視窗進行即時預覽，並透過 stdin/stdout 接收 JSON 指令。
    - **Bridge (Extension Host)**：在 `extension.ts` 中實作 `DatasetSidecarManager`，負責 spawn 進程與中轉訊息。
    - **Frontend (Webview)**：重構 `sampler.js`，將原本的瀏覽器媒體調用替換為抽象指令（如 `datasetCaptureImage`）。
- **優勢**：原生預覽無延遲、不受瀏覽器沙盒限制、方便後續擴充 MediaPipe 特徵提取。

### 2. Sidecar 通訊協定設計
- **指令集**：
    - `startCamera`: 開啟 OpenCV 預覽視窗。
    - `stopCamera`: 關閉視窗並釋放資源。
    - `captureImage`: 擷取當前畫面，回傳 Base64 縮圖與儲存路徑。
    - `exit`: 安全關閉 Sidecar 進程。
- **異步機制**：前端呼叫返回 Promise，透過 `requestId` 在訊息流中精確匹配回調。
