## 🔧 技術深挖 (Technical Deep Dive)

### 1. Sidecar 通訊模式
- **問題**：訓練是長時間執行的任務，需要即時回報進度
- **解決方案**：
  - Sidecar 使用 `send_event` 主動推送 `trainingLog` 事件
  - Extension 的 `onEvent` 回調接收事件並轉發給前端
  - 前端使用 `onTrainingLog` 監聽並可即時顯示日誌

### 2. Tauri 事件路由
- **問題**：Tauri 的 `emit` 是全域廣播，可能影響多視窗
- **解決方案**：
  - 使用 `window.emit_to(&label, ...)` 定向發送（待實作）
  - 目前使用單一視窗，暫時使用 `window.emit`

### 3. 前端非同步流程
- **問題**：訓練流程涉及多個非同步步驟（對話框 → SSH 設定 → 訓練）
- **解決方案**：
  - 使用 async/await 重構按鈕事件處理
  - 將對話框改為 Promise-based
  - 改善程式碼可讀性與錯誤處理
