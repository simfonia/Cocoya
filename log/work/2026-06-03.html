技術深挖 (Technical Deep Dive)

1. 解決 Webview 資源存取封鎖 (401 錯誤)

VS Code Webview 對於工作區外的檔案有嚴格限制。原先僅許可延伸模組目錄，導致存放在其他位置的資料集圖片回傳 401 錯誤。最終解決方案是在 activate 時遍歷邏輯磁碟並加入 localResourceRoots，同時在 CSP 中優化 img-src 以相容 webview.cspSource。

2. 匯出前同步機制 (Pre-export Syncing)

為了解決「匯出後只有 JSON」的問題，Host 端在執行 ZIP 打包前會先執行 copyRecursiveSync。此方法具備增量校驗（檢查檔案大小），能高效地將分散在各處的匯入樣本集中到專案暫存目錄，確保打包後的 ZIP 結構完整。

3. Bridge 通訊層補完

修復了 BaseBridge 缺失 pickFolder 方法的問題。確保前端 UI 透過統一的橋接介面調用原生功能，這符合專案的通訊抽象化原則，也有利於未來向 Tauri 獨立版移植。
