DGX Spark Docker 訓練容器建置指南

完整記錄從檔案準備到模型下載的每一步驟，含踩坑經驗與解決方案

1. 環境需求

1.1 DGX Spark 規格確認

項目規格備註
CPU 架構 linux/arm64 非一般 PC 的 AMD64，所有 Docker 映像須為 ARM64
Docker v24+ 需支援 --gpus all 參數
NVIDIA Driver 535+ GPU 加速必要條件
Python 3.12 DGX Spark 預裝，位置 /usr/bin/python3

1.2 本地端需求（Windows）

項目版本用途
Python3.10+執行自動化腳本
paramiko3.4+SSH/SFTP 連線
tensorflow2.13+本地 TFLite 轉換
📌 關鍵注意 DGX Spark 是 ARM64 架構，不要使用 AMD64 的 TensorFlow 映像。
官方建議使用 nvcr.io/nvidia/pytorch:25.04-py3 作為基礎映像。

2. 檔案準備

2.1 訓練容器模板結構

在 Cocoya 專案中的位置：

resources/dataset_manager/train_templates/classifier/
├── Dockerfile.train ← Docker 映像定義
├── train_classifier.py ← 訓練腳本（容器內執行）
└── requirements.txt ← Python 套件需求

2.2 Dockerfile.train 內容

# ✅ 使用 NVIDIA NGC ARM64 容器 FROM nvcr.io/nvidia/pytorch:25.04-py3 WORKDIR /workspace # 安裝 TensorFlow + 其他套件 RUN pip install --no-cache-dir \ tensorflow==2.16.1 \ scikit-learn \ pillow \ matplotlib COPY train_classifier.py . RUN mkdir -p /output ENTRYPOINT ["python", "train_classifier.py"]
⚠️ 平台選擇關鍵 必須使用 NGC 官方 ARM64 容器，否則會出現 exec format error：
exec /bin/sh: exec format error

2.3 train_classifier.py 核心邏輯

① 讀取資料集 — 從 /dataset 讀取影像（子目錄名 = label）

② 資料擴增 — RandomFlip, Rotation, Zoom, Contrast, Brightness, Translation

③ Class Weight 計算 — sklearn.utils.class_weight 自動平衡

④ MobileNetV2 轉移學習 — ImageNet 預訓練權重

⑤ 訓練 — 支援命令列參數 --epochs, --batch_size, --lr

⑥ 儲存 Keras 模型 — 輸出到 /output/gesture_model.keras

⑦ 儲存 labels.txt — 輸出到 /output/labels.txt

❌ TFLite 轉換在 ARM64 不可行 TensorFlow 2.16 on ARM64 有 keras_deps.get_call_context_function() bug，
容器內無法進行 TFLite 轉換。TFLite 轉換需在本地 PC 進行。

3. 上傳訓練模板到 DGX

3.1 手動上傳（無 SCP 時）

Windows PowerShell 沒有內建 scp，可透過 sftp 手動上傳：

# 啟動 SFTP 連線 sftp your_username@your_dgx_host # 進入 SFTP 後執行： mkdir /tmp/classifier cd /tmp/classifier put resources\dataset_manager\train_templates\classifier\Dockerfile.train put resources\dataset_manager\train_templates\classifier\train_classifier.py put resources\dataset_manager\train_templates\classifier\requirements.txt exit

3.2 使用自動化腳本（04_upload_to_dgx.py）

上傳資料集到 DGX：

# 上傳資料集（要先收集好照片） cd mvp_hand_gesture python 04_upload_to_dgx.py # 依序輸入： DGX Host (e.g. 192.168.1.100): 192.168.3.8 ← 你的 DGX IP SSH Port (22): 22 Username: simfonia Password: ********
✅ 執行流程
  1. 將 dataset/gesture_dataset/ 打包成 gesture_dataset.zip
  2. SFTP 上傳到 DGX 的 ~/cocoya_ai/sessions/{電腦名稱}/dataset/
  3. 遠端自動解壓縮後刪除 ZIP
  4. 本地 ZIP 也自動刪除
⚠️ ZIP 內路徑陷阱 打包時 arcname 要相對於資料集目錄本身，避免產生雙層目錄：
arcname = os.path.relpath(filepath, DATASET_DIR)
arcname = os.path.relpath(filepath, os.path.dirname(DATASET_DIR))

4. 建立 Docker 映像

4.1 在 DGX 上執行

1 SSH 到 DGX

ssh your_username@your_dgx_host

2 進入目錄並確認檔案

cd /tmp/classifier ls -la # 應該看到： # Dockerfile.train train_classifier.py requirements.txt

3 建立 Docker 映像

docker build -t cocoya-train-classifier -f Dockerfile.train . # 注意：檔案名稱是 Dockerfile.train，要用 -f 指定 # 如果改名為 Dockerfile（無副檔名），則不需要 -f
⚠️ 檔案名稱陷阱 Docker 預設找 Dockerfile（無副檔名）。
使用 Dockerfile.train 時必須加上 -f Dockerfile.train 參數。

4.2 Build 輸出範例

# 成功的 build 輸出： [+] Building 5.2s (10/10) FINISHED => [internal] load build definition from Dockerfile.train => [internal] load metadata for nvcr.io/nvidia/pytorch:25.04-py3 => [1/5] FROM nvcr.io/nvidia/pytorch:25.04-py3 => [2/5] RUN pip install tensorflow==2.16.1 scikit-learn pillow matplotlib => [3/5] COPY train_classifier.py . => [4/5] RUN mkdir -p /output => exporting to image => => writing image sha256:919f0acf... => => naming to docker.io/library/cocoya-train-classifier

5. 驗證 Docker 映像

docker images | grep cocoya-train-classifier # 預期輸出： cocoya-train-classifier latest 919f0acf 5.2GB 2 minutes ago
📌 映像大小說明 映像約 5.2GB（含 PyTorch + TensorFlow），這是正常大小。
Docker 映像存在 DGX 本機，建一次即可讓所有學生共用。

6. 執行訓練

6.1 手動執行 Docker 訓練

7. 下載模型到本地

7.1 使用自動化腳本（06_download_from_dgx.py）

cd mvp_hand_gesture python 06_download_from_dgx.py

7.2 SSH 路徑處理（關鍵！）

SFTP 不支援 ~ 路徑展開，必須使用絕對路徑：

# ❌ 錯誤寫法： sftp.stat('~/cocoya_ai/models/model.keras') # FileNotFoundError # ✅ 正確寫法：先取得絕對路徑 stdin, stdout, stderr = ssh.exec_command('echo $HOME') homedir = stdout.read().decode('utf-8').strip() abs_path = f"{homedir}/cocoya_ai/models/model.keras" sftp.stat(abs_path) # 正常運作

7.3 本地模型目錄

# 下載完成後的本地目錄： mvp_hand_gesture/model/ ├── gesture_model.keras (9.3 MB) ← 從 DGX 下載 ├── gesture_model.tflite (8.5 MB) ← 本地轉換的 Float32 版 └── labels.txt (25 B) ← none/paper/rock/scissors

8. 在本地轉換 TFLite

由於 DGX（ARM64）上 TensorFlow 的 TFLite 轉換有 bug，轉換需在本地 PC（AMD64）進行。

python -c " import tensorflow as tf model = tf.keras.models.load_model('model/gesture_model.keras') converter = tf.lite.TFLiteConverter.from_keras_model(model) tflite_model = converter.convert() with open('model/gesture_model.tflite', 'wb') as f: f.write(tflite_model) print(f'TFLite 模型已儲存: model/gesture_model.tflite ({len(tflite_model)/1024:.1f} KB)') "
📌 三個模型的用途
檔案大小用途
gesture_model.keras~9 MB保留用於後續再訓練
gesture_model.tflite~8.5 MB✅ 部署用（Float32 版）

9. 常見問題與解決方案

9.1 Docker Build 失敗

❌ exec /bin/sh: exec format error 原因：使用了 AMD64 的容器映像（如 tensorflow/tensorflow:2.13.0-gpu）
解決：改用 ARM64 映像，如 nvcr.io/nvidia/pytorch:25.04-py3
❌ open /Dockerfile: no such file or directory 原因：檔案名稱是 Dockerfile.train，不是 Dockerfile
解決：加上 -f Dockerfile.train 參數

9.2 TFLite 轉換失敗

❌ 'NoneType' object is not callable 錯誤訊息：
File ".../tflite_keras_util.py", line 188, in _wrapped_model
with keras_deps.get_call_context_function()().enter(...)
TypeError: 'NoneType' object is not callable

原因：TensorFlow 2.16 on ARM64 的 Keras 相容性 bug
解決：在容器內只存 Keras 模型，TFLite 轉換在本地 PC 進行

9.3 SFTP 下載失敗

❌ FileNotFoundError: gesture_model.keras 不存在於遠端 原因：使用 ~ 在 SFTP 路徑中，無法展開
解決：先透過 SSH 取得絕對路徑（echo $HOME），再用絕對路徑進行 SFTP 操作

9.4 主機名稱錯誤

❌ Unknown_PC 路徑 原因：os.uname().nodename 在 Windows 上不存在
解決：改用 socket.gethostname()

9.5 資料集路徑雙層

⚠️ 遠端變成 ~/dataset/gesture_dataset/gesture_dataset/ 原因：ZIP 打包時 arcname 使用了 os.path.dirname(DATASET_DIR)
解決：改為 os.path.relpath(filepath, DATASET_DIR)

9.6 labels.txt 寫入 one-hot 陣列

⚠️ labels.txt 內容變成 [0. 0. 1. 0.] 等陣列 原因：Class Weight 計算的 for 迴圈覆蓋了 labels 變數：
for _, labels in train_ds: # 覆蓋！
解決：改用不同變數名稱 for _, label_batch in train_ds:

10.2 Docker 映像共用原則

✅ 建一次，所有人共用 cocoya-train-classifier 映像只需在 DGX 上建立一次。
所有學生的差異僅在於資料集（透過 -v 掛載不同目錄），而非訓練環境。
# 學生 A 訓練 docker run --gpus all -v /path/A/dataset:/dataset -v /path/A/output:/output cocoya-train-classifier # 學生 B（共用同一映像） docker run --gpus all -v /path/B/dataset:/dataset -v /path/B/output:/output cocoya-train-classifier

10.3 備份/遷移 Docker 映像

# 匯出映像到檔案 docker save cocoya-train-classifier -o cocoya-train-classifier.tar # 在另一台機器載入 docker load -i cocoya-train-classifier.tar
